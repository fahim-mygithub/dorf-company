Draft direction (2026-07-15). Every class has a Class Power — a clickable ability you fire by tapping your dwarf, separate from playing a card. Three layers: the role is a shared substrate (a resource), the class is one Class Power (gated by a cooldown) that spends or triggers it, and a skill tree (post-MVP) upgrades that power. Same engine per role, distinct verb per class. This is the Tank sheet — Support and DPS each have their own sheet now. Numbers are unsimmed placeholders.

🛡️ TANK — the Guard substrate

Guard is a stacking resource every tank builds by playing cards. It soaks like block and pulls threat — enemies already target the tankiest dwarf, so stacking Guard is how a tank volunteers to eat the hit. The cards build the resource; the Class Power spends it, differently per class: the Barbarian trades it for a rage stance (block → damage-resistance), the Fighter cashes a tempo burst, the Paladin discharges banked holy charge. Guard/Devotion live on the party dict, host-side, so both the meter and the power ride the snapshot.

BUILT — draft, unsimmed

🟣 In combat: the Class Power

Each dwarf carries its Class Power as an orb on its portrait — a Hearthstone-style hero power. Tap your own dwarf to fire it; the orb shows the gate (its cooldown, or the resource it spends). This is a second lever beside the hand: cards build the resource, the Class Power spends it. In co-op you only ever fire your own.

The build cost — what Tank adds to combat

The Class Power is the one genuinely new system: a per-dwarf power + power_cd (turns remaining) on the party dict — it rides the snapshot, and the host ticks every cooldown down one at the start of each player phase. A tap-your-portrait input routes to a host-side _fire_power(seat) that checks the cooldown and resolves; in co-op the tap is an intent the host validates, exactly like a card play.

Enrage is a stance, so it needs a bit more: a raging flag (halves incoming damage, blocks gain_guard, adds the Bloodied bonus) and an upkeep check at end of turn — if attacked_this_turn is false, rage drops and power_cd jumps to 4. Action Surge and Divine Smite just set power_cd on use (4 and 3).

New ops: gain_guard v (Guard joins the threat/targeting read) · dmg_per_guard (Fighter Shield Bash) · pay_with_guard (Reforge: the next card’s cost is drawn from Guard, not energy) · X-cost support (cost "X" spends all energy — or all Guard, via Reforge — and Whirlwind deals that much to every enemy) · plus the three power effects (enrage / action_surge / smite). Reused as-is: dmg, block, self_dmg, if_bloodied, draw, gain_energy, temp/retaliate, party_block, party_buff, shield_ally, spend_devotion, force_target_all. Promoting Frenzy / Action Surge / Divine Smite from cards to powers kept the card packs lean.
